# Strong Warning

이 문서는 `air-bridge` 사용 전에 반드시 확인해야 하는 경고 문서다.

영문 문서는 [`warning.en.md`](warning.en.md) 를 참고한다.

## 핵심 경고

`air-bridge` 는 air-gap 환경에서 파일을 이미지 시퀀스로 변환하고 다시 복원하는 도구다. 이 성격 때문에 사용 환경에 따라 보안 정책, 내부 통제, 계약, 법률, 감사 규정과 충돌할 수 있다.

특히 아래 목적에는 사용하면 안 된다.

- 접근 통제, 보안 모니터링, DLP, 감사 체계, 포렌식 체계를 우회하거나 약화하려는 목적
- 소유자나 관리자 승인 없이 제3자 시스템, 조직 자산, 고객 환경, 학교/회사 장비에서 데이터를 반출하거나 반입하려는 목적
- 악성코드 반입, 무단 정보 이동, 비인가 자료 복제, 로그 회피, 추적 회피 목적

이 프로젝트의 `identify`, `pack`, `unpack` 같은 보조 기능도 위와 같은 우회 목적을 정당화하지 않는다. 이 기능들은 승인된 배포 준비와 파일 구성 확인 범위에서만 사용해야 한다.

## 사용 전 확인 사항

사용자는 실제 사용 전에 최소한 아래 항목을 직접 확인해야 한다.

1. 대상 시스템과 네트워크의 소유자 또는 관리자로부터 명시적 승인을 받았는가
2. 적용 환경의 법률, 계약, 내부 정책, 보안 규정을 검토했는가
3. 데이터 반출입 자체가 허용되는 업무 범위인지 확인했는가
4. 운영 로그, 감사 기록, 변경 이력 요구사항을 충족하는가
5. 운영 환경이 아닌 격리된 테스트 환경에서 먼저 검증했는가

위 조건을 충족하지 못하면 사용하지 않는 것이 맞다.

## 책임 범위

이 프로젝트는 있는 그대로 제공된다. 개발자는 사용자의 실제 사용 방식, 적용 환경, 운영 결과, 정책 위반, 법적 분쟁, 데이터 손실, 서비스 장애, 보안 사고에 대해 책임지지 않는다.

실제 사용 여부와 그에 따른 모든 결과에 대한 판단과 책임은 사용자에게 있다.

## 운영 권고

운영 환경에서 사용해야 한다면 아래 기준을 권장한다.

- 사전 승인 문서를 남긴다
- 테스트 환경에서 먼저 검증한다
- 반출입 대상 데이터와 범위를 최소화한다
- 배포 산출물, 실행 시각, 사용 목적을 기록한다
- 조직의 보안 담당자와 운영 담당자에게 사전 공유한다
